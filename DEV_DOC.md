# Screentation — Документация разработчика (Developer Guide)

**Screentation** — это десктопное приложение для быстрой потоковой аннотации скриншотов с возможностью пакетного сохранения в различных форматах (PNG, JPEG, WebP) с использованием Electron, React, и Konva.js.

---

## 🛠️ Архитектура и ключевые компоненты

### 1. Архитектура процессов и IPC
Main и Renderer процессы полностью разделены в целях безопасности (`contextIsolation: true`, `nodeIntegration: false`). Связь осуществляется через preload-скрипт с помощью строго типизированных каналов IPC (описанных в `shared/constants.ts`):

- `clipboard:read-image` — Чтение картинки из буфера обмена.
- `dialog:select-folder` — Вызов нативного окна выбора папки для сохранения.
- `file:save-image` — Конвертация и запись одного скриншота на диск.
- `file:save-all` — Пакетная конвертация и запись всех скриншотов.
- `settings:get` / `settings:set` — Сохранение и чтение пользовательских настроек через `electron-store`.
- `clipboard:new-image` — Отправка нового изображения из Main процесса в Renderer при фокусе окна.

### 2. Интерактивный холст (Konva.js)
Инструмент для рисования реализован в [Canvas.tsx](file:///d:/Develop/screentation/src/renderer/src/components/Canvas.tsx) с использованием `react-konva`:
- **Масштабирование холста**: Скриншот и аннотации масштабируются с помощью хука `useZoomToFit`. Координаты курсора мыши переводятся в исходную систему координат изображения с помощью инвертирования матрицы трансформации: `stage.getAbsoluteTransform().copy().invert().point(pos)`.
- **Рендеринг размытия (BlurRect)**: Реализован в [BlurRect.tsx](file:///d:/Develop/screentation/src/renderer/src/components/shapes/BlurRect.tsx). Для размытия фрагмента используется Konva-фильтр `Konva.Filters.Blur`. Поскольку фильтры в Konva работают на основе растрового кэша, компонент принудительно вызывает `node.cache()` при изменении своих размеров или координат. В качестве заполнителя (`fillPatternImage`) используется то же фоновое изображение, смещенное на отрицательные координаты элемента.
- **Рендеринг умного ластика (EraserRect)**: Реализован в [EraserRect.tsx](file:///d:/Develop/screentation/src/renderer/src/components/shapes/EraserRect.tsx). Ластик работает по принципу клонирования фрагмента изображения (Clone Stamp). Он рисует ту же фоновую картинку внутри выделенной области (`Rect` с `fillPatternImage`), но со смещением `offsetX` и `offsetY` относительно исходных координат `x` и `y` (что задает область-источник). Пользователь может визуально перетаскивать область-источник ("Source") отдельно от целевой области, чтобы найти идеально подходящий фоновый фрагмент и скрыть объект без визуальных дефектов.
- **Рисование стрелочек (Arrow)**: Используется встроенный компонент `Arrow` из `react-konva`. При его перетаскивании в режиме `select` смещение (координаты `x` и `y`) прибавляется к точкам линии (`points`), после чего координаты смещения сбрасываются обратно в `0,0`, чтобы предотвратить рассинхронизацию.

### 3. Пакетный экспорт с аннотациями
Для того чтобы запекать аннотации на неактивных скриншотах без переключения вкладок на клиенте, в рендерере создается массив холстов (для каждого скриншота свой скрытый экземпляр `Canvas` с `display: none`). 
- Метод `exportToDataUrl` временно сбрасывает масштаб `Stage` в исходный (`scale: 1, x: 0, y: 0`), выполняет экспорт через `stage.toDataURL()` в оригинальном разрешении изображения, а затем возвращает холст в исходное состояние.
- Это позволяет мгновенно получать Data URL с наложенными аннотациями для всех скриншотов при пакетном экспорте.

### 4. Мультиязычные хоткеи
Чтобы предотвратить проблемы с неработающими хоткеями при включенной русской/альтернативной раскладке клавиатуры, хук `useHotkeys` анализирует значение `e.code` (например, `KeyV`, `KeyZ`, `KeyS`) вместо `e.key`. Для управляющих клавиш (цифры, `Escape`, `Delete`, `Backspace`) используется стандартный fallback на `e.key`.

### 5. Работа в фоне и системный трей
- При закрытии основного окна (`mainWindow.on('close')`) событие перехватывается, отменяется (`e.preventDefault()`) и окно просто скрывается (`mainWindow.hide()`).
- В трее создается иконка Screentation с контекстным меню ("Open Screentation", "Quit"). При клике или двойном клике по иконке окно восстанавливается и переводится в фокус.
- Полное закрытие приложения происходит при выборе пункта «Quit» в меню трея (устанавливается флаг `isQuitting = true`, после чего вызывается `app.quit()`).

## 📦 Разработка и сборка

### Системные зависимости
Приложение использует библиотеку `sharp` для быстрой и качественной конвертации изображений на стороне главного процесса. Поскольку `sharp` является нативной C++ зависимостью, она компилируется под Electron.
При установке зависимостей автоматически срабатывает скрипт `electron-builder install-app-deps`.

### Команды

1. **Установка зависимостей**:
   ```bash
   npm install
   ```

2. **Запуск в режиме разработки**:
   ```bash
   npm run dev
   ```

3. **Сборка проекта (компиляция typescript и упаковка vite)**:
   ```bash
   npm run build
   ```

4. **Создание Windows-дистрибутива (инсталлятор `.exe`)**:
   ```bash
   npx electron-builder --win
   ```
   Собранный установщик и распакованная версия будут сохранены в директорию `dist/`.

## 📝 Важные примечания по поддержке
- **Обновление размытия**: При добавлении новых фильтров или изменении параметров кэширования в `BlurRect` помните, что перед вызовом `.cache()` необходимо обязательно вызывать `.clearCache()`, иначе Konva продолжит использовать старый рендер из оперативной памяти.
- **Безопасность буфера**: Захват буфера обмена на Windows происходит по событию `focus` окна. Так как буфер может содержать очень большие изображения, чтение оптимизировано через прямой буфер `toPNG()`, предотвращая утечки памяти.
